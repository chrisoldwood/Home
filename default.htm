Binaries & Sources

Sources are available for everything. Binaries will be limited to those applications where I know people won't want to, or can't build it themselves. Increasingly I am also providing debug builds to help diagnose problems on other set-ups. The code is all linked statically and doesn't use the MFC so no DLL version conflicts should arise. The only real dependency is on the common controls DLL (COMCTL32.DLL) - you need at least v4.71 which shipped with IE 4.

The sources are by no means an example of how to write good code. Some of the core libraries like WCL have a long heritage which predates templates and so uses its own containers and string class. My style has also changed somewhat over the years and I'm not necessarily sure it is for the better. At the end of the day you get what you pay for.
The archives include projects and workspaces for Visual C++ 6. The last time I checked, all the code compiled under Visual C++ 7.x as well.
